Story

I just started making this on a whim. I thought it would be a simple three or four day long project, but it turned out to be much more complex. As always, I wanted to make this as perfect as I could. I ended up spending several weeks, working hours on end to make sure no user would ever have to see a broken view of the website. My goal was to make a simple and pretty start page that would greet a user by name and possibly use natural language to tell them about the weather.

Turns out making a website with client- and server-side processing on your own when you're an absolute beginner is very complicated. I tried my best to make PHP and JavaScript work together and complement each other. There may be some ugly code inside, which I will beautify as I go if it makes the page more efficient. After all, it's meant to be used as a homepage. It needs to load quickly.

For the curious

Learning objectives: I started this off as some PHP practice. I thought server-side scripting was something that would be nice to have under my belt, so I set off to learn it. In the process, I learned much, much more than I had planned to. I learned how to use cookies for per-user persistence, I learned how awesome Bootstrap is and I brushed up my HTML, CSS and JavaScript. A lot. I also used the forecast.io API and designed part of the website graphics myself (rest was done by my younger brother), so it was a nice dip in some graphic design as well.

To summarize: I now have a better feel over how things work on the inside.

How it works: The first time a user visits the site, they are greeted by a location prompt. If they don't allow the website to get their location (using HTML5), the server tries to get an approximate location using their IP address. Then they can choose to personalize their website using their name and temperature unit preference. This data is stored in their browser along with their coordinates so that they don't have to do this each time they visit.

The server calculates weather data and sets the background appropriately before the page is downloaded so it is faster for the user. Time and date is calculated client-side.
